== 사용패킷 ==

# Auth 스레드

en_PACKET_CS_GAME_REQ_LOGIN
en_PACKET_CS_GAME_RES_LOGIN

처리 후 게임모드로 전환 

# Game 스레드

en_PACKET_CS_GAME_REQ_ECHO
en_PACKET_CS_GAME_RES_ECHO



== 사용조건 ==

1. 더미는 1개만 실행 시킵니다. 100 ~ 5000명 수준으로 조절.

2. 더미의 루프가 느리게 돌 경우 타임아웃에 걸릴 수 있으므로
   일단 타임아웃 기능을 서버에서 제외 합니다.

3. 더미에 간혹가다가 [Loop Delay...] 이라는 메시지가 출력되었다 사라집니다.
   이는 1개의 스레드가 너무 느리게 돌 경우 1 Loop 의 지연시간입니다.
   이로 인해서 서버측에 타임아웃이 발생할 수 있습니다.   

4. Auth, Game 스레드의 FPS 를 항시 확인 합니다. 0 으로 떨어질시 더미 인원,수치 조절필요.

5. CPU 가 100% 상태로는 테스트 하지 않습니다. (제대로 된 테스트가 안됨)


== 문제확인 ==

1. 일정시간 테스트 후 접속끊김이나 기타 오류상황은 없는지 확인 합니다.

2. 더미를 끈 후 게임서버 쪽에 패킷반환,접속세션이 모두 정리 되는지 확인 합니다.

3. 더미를 돌리다가 S 키를 눌러서 일시정지를 시키고
   ReplyWait 수치가 0 으로 떨어지는지 확인 합니다.

4. Connect 과정에서 채팅서버처럼 ClientDown 이 잘못 감지되는 경우는 거의 없습니다만
   혹시나 확인이 필요한 경우는 S 를 눌러서 ConnectTotal 수치를 확인 합니다.
